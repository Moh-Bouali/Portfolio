Tournament Management Application

My first software project was during my 1st year, 2nd semester to be more precise. I was tasked to make a tournament management system with C# using .NET framework with a login feature as well as CRUD features. In addition to that there was some logic such as the rules of the games must be different depending on which type of tournament you play. For example, in a tennis tournament you cannot have a score of 99-97, therefore the system makes sure that the scoring is according to the game that is being played. In addition to that, the scheduling of games is done in a round robin way, for that I had to use a round robin algorithim. Finally, there was also a leaderboard feature that would show the leaderboard of players from best to worst.

The goal of this year was to understand the basics of coding and to get to know what are SOLID principles and what clean code really is. In addition to applying simple algorithims

Food delivery application (BACKEND)

In my second project which was during my 2nd year, 3rd semester to be precise. I had to choose my own topic, I decided to make an application that is similar to Thuisbezorgd/Uber eats, where users can login and search for their favourite restaurant and make an order. For that I used Java with Spring Boot framework. There were features such as Authentication and Authorization using Spring security in order to generate JWT tokens, SOLID principles had to be respected with proper layering of the application and websockets had to be implemented. In addition ofcourse to a database connection in order to store and retrieve data, for that I used Mariadb database as a docker image

The goal of this assignment was to understand the fundamentals of creating a full stack web application with SOLID princples. In addition to being able to dockerize the application and to run it through a CI/CD pipeline where all the tests are ran automatically in addition to sonarqube which is a tool that checks the quality of the code.

Internship at Vanderlande

I will not be able to share all the details of what I worked on nor the code due to my non-disclosure agreement with Vanderlande, but what I can say is the following:
This internship was mostly frontend related, I used Angular JS in order to develop a UI to test a concept that the company has had for some time, by developing this UI I was also tasked to test the concept of it with a certain department within Vanderlande as it was supposed to be for them to make their work easier and more effecient, then I had to forward my findings to Vanderlande and advise them in how to proceed.
